prompt 1: 
Create a modern, premium UI for an AI-powered social media assistant called **SocialFlow AI**.

SocialFlow AI is an AI-powered chatbot (AI Agent) that allows users to manage their social media accounts using natural language. Users simply chat with the AI, and it can generate posts, publish content, schedule posts, manage comments, reply to DMs, and perform other social media actions using **Zenrio's API**.

The application follows this workflow:

1. The user sends a message.
2. The AI Agent analyzes the request.
3. If an action is required, the AI Agent generates the appropriate action.
4. The action is executed using Zenrio's API.
5. The execution result is passed to a second AI.
6. The second AI generates a natural, user-friendly response explaining the result.
7. If no action is detected, the second AI simply responds normally without calling any tools.

We use two AI models:

* **AI Agent** – Responsible for understanding the user's request and generating tool actions.
* **Response AI** – Responsible for generating the final conversational response after the action has been completed, or responding normally when no action is required.

For AI models, we will use **OpenRouter's free models**.

For social media management, we will use **Zenrio's API**.

Before building the UI, first study Zenrio's documentation carefully so the interface can support all of its available capabilities, including but not limited to:

* Create Posts
* Schedule Posts
* Publish Posts
* Manage Comments
* Reply to DMs
* Manage Connected Accounts
* Any other supported Zenrio features

This is **not** a full-stack application.

There is:

* No backend
* No database
* No authentication server

All user data, settings, conversations, API keys, and application state should be stored locally using **Local Storage**.

For now, focus **only on building the UI**.

Create the following pages:

* **Chat Page** – Displays all conversations.
* **Chat Window Page** – The main AI chat interface where users interact with the AI assistant.
* **Accounts Page** – Displays all social media accounts connected through Zenrio. These accounts will be fetched from Zenrio's API (e.g., Instagram, LinkedIn, X, Facebook, etc.).
* **Settings Page** – Users must configure everything required before using the application, including:

  * OpenRouter API Key
  * OpenRouter Model Selection
  * Custom Model Input
  * Zenrio API Key
* **Landing Page**
* **Login Page**
* **Sign Up Page**
* **About Page**

UI Requirements:

* Clean, modern, and professional SaaS interface.
* Default dark theme inspired by **shadcn/ui**.
* Primarily use black, white, and gray with subtle accent colors only where necessary.
* Premium spacing and typography.
* Beautiful hover effects, transitions, and micro-interactions.
* Smooth animations using Framer Motion where appropriate.
* Responsive across desktop, tablet, and mobile.
* Maintain a consistent design system throughout the application.

Do not build a generic admin dashboard. The application should feel like a premium AI product similar in quality to ChatGPT, Linear, and Vercel, with the **AI chat experience as the primary focus**.
